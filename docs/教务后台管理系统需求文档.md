# 高中教务后台管理系统需求文档

## 1. 项目定位

本项目是一个用于学习和练习的高中教务后台管理系统 demo，主要面向单所高中内部使用。

系统第一版优先实现学生档案管理，后续再逐步扩展班级、教师、课程、考勤、成绩、通知公告、数据分析和系统权限等模块。

后端暂不接入真实数据库，优先使用 `json-server` 模拟接口和数据。设计时需要保证数据结构清晰、接口简单、便于以后替换为真实后端服务。

## 2. 用户角色

系统后续规划支持以下角色：

- 校长
- 教务主任
- 班主任
- 任课教师
- 学生
- 家长
- 系统管理员

第一版暂不实现完整账号体系和权限体系，先以后台管理视角完成学生档案相关功能。

## 7. json-server 数据模拟设计

### 7.1 设计原则

由于项目是学习 demo，后端使用 `json-server` 模拟数据是可行的。需要注意以下几点：

- 数据尽量扁平化，避免过深嵌套。
- 常用查询字段直接放在学生对象上，方便列表筛选。
- 关联数据使用 id 关联，例如 `classId`、`gradeId`。
- 第一版不做复杂事务，只在前端操作成功后同步更新相关字段。
- 接口命名尽量接近真实 REST 风格，便于以后替换真实后端。

### 7.2 建议数据表

`db.json` 中建议包含：

- students
- parents
- grades
- classes
- academicYears
- semesters
- studentStatusChanges
- dormitories
- dictionaries

第一版最少需要：

- students
- parents
- grades
- classes
- academicYears
- semesters
- studentStatusChanges

## 8. 页面风格要求

系统整体风格参考企业 SaaS 后台：

- 页面清爽、专业、信息密度适中。
- 使用侧边栏导航和顶部工具栏。
- 表格、筛选、抽屉、弹窗、标签页作为主要交互方式。
- 重要状态使用清晰的标签展示。
- 学生详情页强调信息归档感，不做花哨装饰。
- 首页和数据页可以适当使用统计卡片和图表。

## 12. 技术栈约定

### 12.1 前端技术栈

项目第一阶段固定使用以下技术栈：

- Vue 3：前端主框架
- TypeScript：类型约束
- Vite：开发和构建工具
- Vue Router：页面路由
- Pinia：状态管理
- Element Plus：后台管理系统 UI 组件库
- @element-plus/icons-vue：图标库
- Axios：HTTP 请求
- Dayjs：日期处理
- json-server：模拟后端 REST API
- concurrently：同时启动前端服务和模拟接口服务

### 12.2 包管理约定

项目统一使用 `pnpm` 管理依赖。

后续新增、删除或更新依赖时，统一使用：

```bash
pnpm add 包名
pnpm add -D 包名
pnpm remove 包名
```

不再使用 `npm install` 或 `yarn add`。

### 12.3 后端模拟约定

本项目暂不开发真实后端服务，使用 `json-server` 模拟接口。

设计原则：

- 使用 `db.json` 存放模拟数据。
- 接口尽量保持 REST 风格。
- 数据结构尽量扁平，便于筛选、分页和后续替换真实后端。
- 第一版不处理复杂权限、事务和真实审批流。
