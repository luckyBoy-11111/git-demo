# 系统目录结构说明

本文档用于说明教务后台管理系统前端项目的目录职责。当前项目基于 Vue 3、TypeScript、Vite、Vue Router、Pinia、Element Plus 和 json-server 设计。

## 设计原则

- 按业务模块拆分页面、接口、类型和局部组件。
- 通用能力统一放入 `shared`，避免散落在业务模块中。
- 布局、路由、状态初始化等应用级能力放入 `app` 和 `layouts`。
- 第一阶段重点开发 `students` 模块，其他模块先保留扩展目录。
- 模拟数据统一放入 `src/mock` 或根目录 `db.json`，便于后续替换真实后端。

## 根目录

```text
docs/
src/
public/
package.json
vite.config.ts
tsconfig.json
```

### `docs`

项目文档目录，用于存放需求补充、目录说明、接口说明、开发约定等文档。

建议后续可继续增加：

- `api-design.md`：接口设计说明。
- `development-guide.md`：开发规范和启动方式。
- `mock-data.md`：模拟数据结构说明。

### `src`

前端源码主目录，所有应用代码、页面、组件、类型、样式和模拟数据都放在这里。

### `public`

静态资源目录。适合放不经过 Vite 打包处理、需要按固定路径访问的文件，例如 favicon、静态 SVG、第三方静态文件等。

## `src/app`

应用级基础设施目录，负责项目启动、路由、全局状态、全局样式等能力。

```text
src/app/
  router/
  store/
  styles/
```

### `src/app/router`

路由配置目录。

建议职责：

- 定义系统路由表。
- 配置路由守卫。
- 维护模块路由的聚合入口。

推荐文件：

- `index.ts`：创建并导出 Vue Router 实例。
- `routes.ts`：维护主路由配置。

### `src/app/store`

全局状态初始化目录。

建议职责：

- 创建并导出 Pinia 实例。
- 放置应用级 store 的聚合入口。

业务模块自己的 store 应优先放在对应模块目录下，例如 `src/modules/students/stores`。

### `src/app/styles`

全局样式目录。

建议职责：

- 全局 CSS 变量。
- Element Plus 样式覆盖。
- 通用基础样式。
- 主题相关样式。

## `src/layouts`

页面布局目录，主要负责后台系统的整体框架。

```text
src/layouts/
  components/
```

### `src/layouts`

适合放系统级布局组件。

推荐文件：

- `AdminLayout.vue`：后台管理主布局，包含侧边栏、顶部栏、内容区。

### `src/layouts/components`

布局内部使用的组件目录。

建议组件：

- `AppSidebar.vue`：左侧菜单。
- `AppHeader.vue`：顶部工具栏。
- `AppBreadcrumb.vue`：面包屑。
- `AppTabs.vue`：页面标签栏，后续需要时再实现。

## `src/modules`

业务模块目录。每个一级业务模块独立维护自己的页面、接口、类型、组件和局部状态。

```text
src/modules/
  dashboard/
  students/
  classes/
  teachers/
  courses/
  schedules/
  attendance/
  exams/
  notices/
  analytics/
  system/
```

### `src/modules/dashboard`

工作台模块。

当前需求中第一版暂不开发，后续用于展示学校概览、教务待办、今日课程、异常提醒等内容。

### `src/modules/students`

学生管理模块，是第一阶段核心模块。

```text
src/modules/students/
  api/
  components/
  pages/
  stores/
  types/
  utils/
```

#### `src/modules/students/api`

学生模块接口目录。

建议职责：

- 学生档案接口。
- 家长信息接口。
- 学籍异动接口。
- 住宿信息相关接口。

推荐文件：

- `student.api.ts`
- `parent.api.ts`
- `status-change.api.ts`

#### `src/modules/students/components`

学生模块内部组件目录，只服务于学生模块。

建议组件：

- `StudentSearchForm.vue`：学生列表筛选表单。
- `StudentTable.vue`：学生列表表格。
- `StudentForm.vue`：新增和编辑学生表单。
- `StudentProfileCard.vue`：学生详情顶部档案卡片。
- `ParentInfoPanel.vue`：家长信息面板。
- `BoardingInfoPanel.vue`：住宿信息面板。
- `StatusChangeDialog.vue`：学籍异动弹窗。

如果组件被多个模块复用，应移动到 `src/shared/components`。

#### `src/modules/students/pages`

学生模块页面目录。

建议页面：

- `StudentListPage.vue`：学生列表页。
- `StudentCreatePage.vue`：新增学生页。
- `StudentEditPage.vue`：编辑学生页。
- `StudentDetailPage.vue`：学生详情页。

#### `src/modules/students/stores`

学生模块 Pinia 状态目录。

适合放学生列表筛选条件、当前学生详情、模块级缓存等状态。不要放纯展示组件自己的局部状态。

#### `src/modules/students/types`

学生模块类型定义目录。

建议文件：

- `student.ts`：学生档案类型。
- `parent.ts`：家长信息类型。
- `status-change.ts`：学籍异动类型。

#### `src/modules/students/utils`

学生模块工具方法目录。

适合放：

- 学生状态格式化。
- 表单数据转换。
- 接口数据到页面数据的 mapper。
- 学生模块下拉选项配置。

### `src/modules/classes`

班级管理模块。

后续用于年级管理、班级管理、班级学生、班主任配置、任课教师配置等功能。

### `src/modules/teachers`

教师管理模块。

后续用于教师档案、任课科目、所带班级、教师课表、工作量统计等功能。

### `src/modules/courses`

课程管理模块。

后续用于科目管理、年级课程配置、教学计划、教学进度等功能。

### `src/modules/schedules`

排课管理模块。

后续用于学期课表、班级课表、教师课表、调课、停课、代课和冲突检测等功能。

### `src/modules/attendance`

考勤请假模块。

后续用于学生考勤、教师考勤、请假申请、请假审批和缺勤统计等功能。

### `src/modules/exams`

考试成绩模块。

后续用于考试计划、成绩录入、成绩查询、成绩分析和排名等功能。

### `src/modules/notices`

通知公告模块。

后续用于校园公告、班级通知、家长通知、已读未读统计和消息模板等功能。

### `src/modules/analytics`

数据分析模块。

后续用于校长驾驶舱、学生数据分析、班级数据分析、成绩趋势分析和出勤分析等功能。

### `src/modules/system`

系统设置模块。

后续用于学年学期、用户账号、角色权限、数据字典、操作日志和学校基础信息等功能。

## `src/shared`

跨模块共享能力目录。只有多个模块都会使用的代码才放在这里。

```text
src/shared/
  api/
  components/
  constants/
  hooks/
  types/
  utils/
```

### `src/shared/api`

通用接口基础能力目录。

建议职责：

- Axios 实例。
- 请求和响应拦截器。
- 通用错误处理。
- API 基础地址配置。

推荐文件：

- `http.ts`
- `endpoints.ts`

### `src/shared/components`

跨模块复用组件目录。

适合放：

- 页面标题组件。
- 通用表格封装。
- 表单分组组件。
- 状态标签组件。
- 通用确认弹窗。

如果组件只服务某个业务模块，应优先放在对应模块的 `components` 下。

### `src/shared/constants`

全局常量目录。

适合放：

- 菜单配置。
- 通用字典。
- 固定枚举。
- 全局配置项。

### `src/shared/hooks`

组合式函数目录。

适合放：

- 分页逻辑。
- 表格加载状态。
- 字典数据读取。
- 通用查询逻辑。

模块强相关的组合式函数可以放在对应模块内部。

### `src/shared/types`

全局通用类型目录。

适合放：

- 分页类型。
- 接口响应类型。
- 通用选项类型。
- 菜单类型。

业务实体类型不要放这里，应放在对应业务模块的 `types` 下。

### `src/shared/utils`

通用工具函数目录。

适合放：

- 日期格式化。
- 字符串格式化。
- 通用校验函数。
- 空值处理。

## `src/mock`

模拟数据目录。

建议职责：

- 存放 json-server 使用的数据文件或 mock 辅助说明。
- 第一阶段可放 `db.json`。

如果 json-server 启动命令要求根目录 `db.json` 更方便，也可以将主数据文件放在项目根目录，并在 `src/mock` 中保留 mock 数据说明或拆分源文件。

## `src/assets`

项目资源目录。

```text
src/assets/
  images/
  icons/
```

### `src/assets/images`

图片资源目录，例如学生默认头像、页面插图、空状态图片等。

### `src/assets/icons`

图标资源目录。项目已使用 `@element-plus/icons-vue`，所以这里主要用于放业务专属 SVG 或无法由组件库覆盖的图标。

## 目录使用约定

### 页面放哪里

页面组件放在对应模块的 `pages` 下。

示例：

```text
src/modules/students/pages/StudentListPage.vue
```

### 业务组件放哪里

只属于一个业务模块的组件，放到该模块的 `components` 下。

示例：

```text
src/modules/students/components/StudentTable.vue
```

多个模块复用的组件，放到 `shared/components`。

### 接口文件放哪里

业务接口放到对应模块的 `api` 下。

示例：

```text
src/modules/students/api/student.api.ts
```

Axios 实例和请求基础封装放到：

```text
src/shared/api/http.ts
```

### 类型文件放哪里

业务实体类型放到对应模块的 `types` 下。

示例：

```text
src/modules/students/types/student.ts
```

跨模块通用类型放到：

```text
src/shared/types
```

### 状态管理放哪里

业务模块状态放到对应模块的 `stores` 下。

示例：

```text
src/modules/students/stores/student.store.ts
```

Pinia 初始化放到：

```text
src/app/store
```

### 样式放哪里

全局样式、主题变量和 Element Plus 覆盖放到：

```text
src/app/styles
```

组件局部样式优先写在对应 `.vue` 文件中。

## 第一阶段推荐实现顺序

1. 完成 `shared/api/http.ts` 和 json-server 请求基础封装。
2. 完成 `app/router` 路由结构。
3. 完成 `layouts/AdminLayout.vue` 后台基础布局。
4. 完成 `students/types` 学生、家长、学籍异动类型。
5. 完成 `students/api` 学生模块接口。
6. 完成学生列表、学生表单、学生详情页面。
7. 补充 `db.json` 模拟数据。
